# 용어정리
- 명령 라인 인터페이스 :
코드나 단어 등 명령어 입력을 키보드로 요구하는 형태이다.

- GUI :
사용하기 편리한 사용자 인터페이스로 아이콘 형태를 이용하여 자유로운 마우스 사용을 허용한다.


- 프로그램과 프로세스
	1) 프로그램의 정의
		- 특정 목적을 수행하기 위해 나열된 작업의 목록(명령어 목록)은 프로그램이다.	
		- 컴퓨터에서 프로그램은 프로그래밍 언어로 작성된 작업 수행 과정이다.

	2) 프로세스의 정의
		프로세스는 "메모리에 적재되어 프로세서(cpu)에 의해 실행중인 프로그램"

	즉, 작업의 과정이 파일로 저장되어 있으면 그것을 "프로그램"이라고 부르고 
	메모리에 적재되어 실행 중 이거나 실행이 대기 중일 땐 "프로세스"라고 구별하여 부를 뿐이다.

- 프로세서(Processor)와 프로세스 :
[출처 : https://blogger.pe.kr/422 ]
	1) 프로세서 정의 :
		- 프로세서는 하드웨어적 측면에서 " 컴퓨터 내에서 프로그램을 수행하는 하드웨어 유닛"이다.
		- 이는 중앙저리장치(CPU)를 뜻하며, 
		폰노이만 아키텍쳐에 의해 만들어졌다면 적어도 하나 이상의 
		산술 논리 연산장치(ALU, Arithmetic Logic Unit)와 
		처리 레지스터(Register)를 내장하고 있어야 한다.
		
	2) 프로세스 정의 :		
		- 프로세스는 "메모리에 적재되어 프로세서(cpu)에 의해 실행중인 [프로그램]"
			> 프로그램은 수행 해야하는 명령어들의 목록(문서)이다.

	**결론 : 프로세스는 "메모리에 적재되어 프로세서(cpu)에 의해 실행중인 프로그램"

		>>> 간단한 "프로세서(cpu)에 의해 실행중인 프로그램(명령어 목록)" 실행 과정
		1. GUI나 명령어를 통하여 프로그램을 실행한다.
		2. 보조기억장치(ssd)에 파일로 저장되어 있던 데이터가 로더(램의기능)에 의해 주기억장치(RAM)에 적재된다.
		3. 주기억장치에 적재된 명령어(코드)중 당장 사용될 일부 데이터를 캐시메모리에 적재한다.
		4. 실행한 프로그램에서 처음으로 실행해야 할 명령어(기계어코드)가 저장된 메모리 주소를 
		프로세서(CPU)의 명령주소(IP: Instruction Pointer) 레지스터에 저장한다.
		5. 프로세서(CPU)는 IP 레지스터가 가리키는 메모리 주소에서 실행할 명령어를 
		인출(메모리에서 CPU로 가져오는것)하여 명령 레지스터(IR : Instruction Register)에 저장한다
		6. IR에 저장된 명령을 실행하고 IP에 다음번에 실행할 명령어가 있는 주소를 저장한다.
		7. [5~6]을 프로그램 끝까지 반복한다.
	
	
- 레지스터(Register) :
	- CPU의 3가지 기능 ALU, CU, Register 중 하나
	레지스터(Register)는 CPU 내부에서 처리할 명령어나 연산의 중간 결과값 등을 
	일시적으로 기억하는 임시 저장소이다.


- 산술 논리 연산장치(ALU, Arithmetic Logic Unit) :
	- CPU에서 실제로 연산 등을 수행하는 장치(유닛)을 말한다.
	- 주로 산술 연산(가감승제), 논리연산(OR, AND, NOT)을 하며, 
	추가적으로, 보수연산, 시프트 연산 등을 수행한다.
	

- ROM
	바이오스 프로그램이 공정과정에 설치되어 있다.

# 운영체제의 기능

## 자원관리 기능
-  운영체제는 [메모리 관리, 프로세스 관리, 주변장치 관리, 파일 관리] 4가지 자원을 관리한다.

1. 메모리 자원 관리기능
	
	1) 메인 메모리(RAM : 주기억 장치) 관리 :	
	- 프로세서가 직접 주소로 지정할 수 있는 유일한 메모리
	
	- 메인 메모리 관리 기능(=  RAM 메모리 관리 기능)	
		> 메모리의 어느 부분을 사용하고, 누가 사용하는지 점검
		> 메모리에 저장할 프로세스 결정
		> 메모리를 할당하고 회수하는 방법 결정
		즉, 프로세스들이 제한된 메모리 영역을 사용할때 효율적으로 사용하게 하는 기능이다.
		
	2) 보조기억장치 관리 
	- 메인 메모리 공간이 제한되어 데이터와 프로그램을 계속(무한히) 저장할 수 없어 보조기억장치 사용
	
	- 보조기억장치 관리의 기능
		> 보조기억장치의 빈 여유 공간 관리
		> 새로운 파일 작성 시 저장 장소 할당
		> 메모리 접근 요청 스케줄링	// 보조기억장치가 주기억장치에 데이터를 넣는 스케줄을 운영체제가 관리한다.
		> 파일 생성하고 삭제
	
2. 프로세스 자원 관리기능
	- 하나의 [프로세스]를 실행하기 위해서는 프로세서(CPU), 메모리, 파일, 입출력 장치와 같은 자원이 필요하다.
		*프로세스 : 하나의 실행중인 프로그램, 하나의 실행중인 작업, 실행중인 명령어 목록을 말한다.
	- 위와 같은 자원은 프로세스를 생성(프로그램을 실행)할 때 운영체제에 의해 프로세서(cpu)가 제공한다. (실행중에도 할당이 가능하다.)
	
	1) 시스템
	- 프로세스의 집합
	- 시스템 코드 수행하는 운영체제 프로세스와 
	사용자 코드를 수행하는 사용자 프로세스(응용프로그램)로 구분한다.
	- 모든 프로세스는 프로세서(cpu)를 분할 사용하여 병행 작업(수행)이 가능하다

	2) 프로세스를 관리 하기 위한 운영체체가 하는 기능
	- 프로세스와 [스레드] 스케줄링
		*스레드 : 프로세스 안의 작업단위
	- 사용자 프로세스(응용 프로그램)와 시스템 프로세스 생성, 제거
	- 프로세스 중지, 재수행
	- 프로세스 동기화 방법 제공
	- 프로세스 통신 방법 제공
	- 교착상태(Deadlock)를 방지하는 방법 제공
	프로세스가 동시에 실행되어 이도저도 못하는 상황
	
3. 주변장치 관리기능
	1) 입출력 장치 관리	
	- 운영체제는 특수 프로그램인 '장치 드라이브'를 사용하여 입출력장치와 상호작용
	- 장치 드라이버는 특정 하드웨어장치와 통신할 수 있는 인터페이스를 제공하므로 
	특정 하드웨어에 종속된 프로그램	
	- 임시 저장buffer-caching 시스템 기능 제공 
	- 일반 장치용 드라이버 인터페이스 제공 
	- 특정 장치 드라이버 제공

4. 파일 관리
	1) 운영체제가 파일 관리를 위해 하고 있는 일
	- 입출력 파일의 위치, 저장, 검색 관리를 의미	
	- 컴퓨터 시스템은 물리적으로 다양한 형태로 파일을 저장 가능하다.
	- 운영체제는 데이터의 효율적 사용을 위해 단일화된 저장 형태 제공
	- 운영체제는 파일의 용이한 사용을 위해 보통 디렉터리로 구성, 
	다수의 사용자가 여기에 접근하려고 할 떄는 이 접근을 제어한다.

	2) 파일 관리를 위한 운영체제의 기능
	- 파일 생성, 삭제
	- 디렉터리 생성, 삭제
	- 보조기억장치의 파일 맵핑
	- 안전한(비휘발성) 저장장치에 파일 저장 	// 주기억 장치(RAM)은 휘발성이다.ㄴ
	
## 시스템 보호
- 보호란? 컴퓨터 자원에서 프로그래, 프로세스, 사용자의 접근 제어 방법
- 운영체제는 파일 사용 권한 부여, 데이터 암호화 등 서비스를 제공, 데이터와 시스템을 보안한다.
- 컴퓨터 시스템에서는 여러 프로세스 동시 실행이 가능하므로 상호 보호를 해야한다.
- 윈도우에서는 랜섬웨어나 바이러스로 인해 폴더에 영향을 주었을때 
파일을 복구할 수 있는 시스템 보호 기능을 지원한다.		// 파일 폴더의 이전 버전을 보관하고 있어 가져올수 있다,


## 네트워크(통신 기능)
- 프로세서는 다양한 방법으로 구성된 네트워크 이용, 완전 접속과 부분 접속 방법으로 연결
- 연결된 프로세서가 통신을 할 때는 경로 설정, 접속 정책, 충돌, 보안 등 고려(운영체제가 관리)


# 운영체제 서비스

1. 부팅(Booting) 또는 부트스트래핑(Bootstrapping)
	- 운영체제를 메인 메모리에 적재하는 과정
	- 부트 로더는 부트스트랩 로더(Bootstrap loader) 줄인 말로 하드디스크와 같은 
	보조기억장치에 저장된 운영체제를 메인 메모리에 적재하는 ROM에 고정(적재)시킨 소규모 프로그램	
		> 사용자 프로그램은 운영체제가 자원을 할당하여 실행시킨다.
		> 컴퓨터 실행시 ROM에 바이오스 프로그램에 의해 
		운영체제가 실행되는 과정
			1) 메인보드의 ROM에 있는 프로그램(바이오스 프로그램)에 의해 
			보조기억 장치에 있는 운영체제의 '부팅 프로그램'이 메인 메모리에 로딩된다.
			2) 부팅 프로그램에 의해 운영체제가 주기억 장치에 로딩이 된다.
			3) 실행된 운영체제에 의해 시작프로그램 목록에 있는 사용자 프로그램들이 실행이 된다.
		
			**참고 : 운영체제 1주 2차 01.jpg	
2. 사용자 서비스 제공
	
	1) 사용자 인터페이스 서비스
	- 사용자와 컴퓨터 간의 상호작용 발생 공간 (CLI, 메뉴, GUI 등 구현) 
		> CLI(Command Line Interface, 명령 라인 인터 페이스) 
		> 사용자가 키보드 등으로 명령어 입력하여 시스템에서 응답 받은 후, 
		또 다른 명령어를 입력 하여 시스템을 동작하게 하는 텍스트 전용 인터페이스
	2) 메뉴 인터페이스 서비스
	- 메뉴 등을 사용하여 시스템과 상호작용
	- 사용 매우 편리, 배우거나 기억해야 할 명령 없음
	
	3) GUI(Graphical User Interface, 그래픽 사용자 인 터페이스) 서비스
	- 윈도우 환경에서 사용자에게 정보와 작업을 표현하는 텍스트, 레이블이나 텍스트 탐색과 함께 
	그래픽 아이콘과 시각적 표시기, 버튼이나 스크롤바와 같은 위젯(Widget) 그래픽 제어 요소를 사용

	4) 프로그램 실행 서비스
	- 프로그램 실행하려면 먼저 메모리에 적재, 프로세서 시간 할당
	- 운영체제는 프로그램을 실행하려고 메모리 할당이나 해제, 스케줄링 등 중요 작업 처리
	
	5) 입출력 동작 수행
	- 운영체제는 입출력 동작 직접 수행할 수 없는 사용자 프로그램의 입출력 동작 방법 제공

	6) 파일 시스템 조작
	- 사용자는 디스크에서 파일 열고, 저장, 삭제하는 등 다양하게 파일 조작

	7) 통신(네트워크)		// 통신은 네트워크 뿐만아니라  프로세서들 간의 정보 교환도 의미
	- 프로세스가 다른 프로세스와 정보를 교환하는 방법
	- 동일한 컴퓨터에서 수행하는 프로세스 간의 정보 교환
	- 두 번째는 네트워크로 연결된 컴퓨터 시스템에 서 수행하는 프로세스 간의 정보 교환	
	
	8) 오류 탐지
	- 운영체제는 가능한 모든 하드웨어와 소프트웨어 수준에서 오류 탐지, 
	시스템 모니터링하여 조정 함으로써 하드웨어 문제 예방
	- 입출력 장치에 관련된 오류와 메모리 오버 플로, 하드디스크의 불량 섹터 검출, 
	부적당한 메모리 접근과 데이터 손상 등
	- 운영체제는 다음 오류 유형을 감지한 후 유형별 로 적절히 조치

3. 시스템 서비스
	1) 자원 할당
	- 운영체제는 다수의 사용자나 작업 동시 실행 시 운영체제가 자원을 각각 할당하도록 관리
	- 프로세서 사이클, 메인 메모리, 파일 저장 장치 등은 특수한 할당 코드를 갖지만, 
	입출력장치 등 더 일반적인 요청과 해제 코드 가질 수 있음
	
	2) 계정 
	- 운영체제는 각 사용자가 어떤 컴퓨터 자원을 얼마나 많이 사용하는지 정보 저장 추적
	- 이 정보는 사용자 서비스 개선을 위해 시스템 재구성하는 연구자에게 귀중한 도구가 됨

	2) 보호와 보안
	- 운영체제는 다중 사용자 컴퓨터 시스템에 저장 된 정보 소유자의 사용을 제한
	- 서로 관련이 없는 여러 작업을 동시에 수행할 때 는 한 작업이 다른 작업이나 운영체제를 방해하지 못하게 해야 함
	
	*보호란? :
		시스템 호출 하려고 전달한 모든 매개변수의 타 당성 검사하고, 
		시스템 자원에 모든 사용자 접근 을 제어하도록 보장하는 것 
	
	*보안이란? : 
		잘못된 접근 시도에서 외부 입출력장치 방어, 
		외부에 사용자 인증을 요구하는 것 

4. 시스템 호출 서비스(운영체제가 제공하는 API)
	- 실행 중인 프로그램과 운영체제 간의 인터페이스, API(Application Programming Interfaces)라고도 함
	- 사용자 프로그램은 시스템 호출을 하여 운영체제의 기능 제공 받음
	- 핵심 커널 서비스와 통신, 새로운 프로세스의 생성 과 실행, 하드웨어 관련 서비스 등이 있음
	- 시스템과 상호작용하는 동작은 대개 사용자 수준 프로세스에서는 사용할 수 없으나, 
	시스템 호출을 하여 운영체제에 서비스를 요청할 수 있음
	
	> 즉, 사용자 프로그램(응용프로그램)은 시스템 프로그램을 직접 사용할 수는 없으나
	운영체제가 제공하는 시스템 호출 서비스(API)를 통하여 운영체제에 서비스를 요청할 수 있다.
	
	1) 시스템 호출 방법
	- 프로그램에서 명령이나 서브루틴의 호출 형태로 호출	
	- 시스템에서 명령 해석기를 사용하여 대화 형태로 호출

	2) 운영체제가 제공하는 일반적인 시스템 호출(API)
	- 프로세스 제어, 파일 조작, 장치 관리, 정보 유지 등

# 운영체제의 미래

1. 사람 같은 인공지능 OS , 미래 아닌 현재형

1) 아멜리아
- 월스트리트저널은 영화 ‘그녀(Her)’의 여주인공인 인공 지능(AI) OS ‘사만다’와 가장 비슷한 존재로 
‘아멜리아’ 를 소개하는 기사를 지난 9월 28일 보도함

- 기사에 의하면 통신·에너지분야 등 다양한 산업의 포춘 100대 기업들은 이미 
아멜리아를 테스트 중에 있음

- 아멜리아는 새로운 인공지능 접근법인 ‘인지 컴퓨팅 (Cognitive computing)’을 적용한 시스템

- 인지 컴퓨팅의 가장 큰 특징은 학습이 가능한 기계라는 것
	>  미국 IT 서비스 회사 IP소프트가 개발 중에 있는 아멜리아는 
	교과서·대화 기록·이메일 등 모든 종류의 텍스트로부터 배움
	> 아멜리아가 받아들인 데이터에 답이 들어있는 한 OS는 문제를 해결할 수 있는 능력을 지니게 됨.
	* 중요한 점은 "답이 들어있는 데이터가 있는 한 문제를 해결할 수 있다." 이다  AI는 아니다.

2. 그 이외의 미래 운영체제
1) 구글 OS 푸크시아
	- 구글이 크롬에 이어 세번째 OS인 '푸크시아'를 최근 개발자들에게 공개함
	- 초기버전이기 때문에 앞으로 변경될 가능성이 많지만, IoT 및 ATM기 등 
	확장성을 염두한 것이라는 평 가가 나오면서 관심이 높아지고 있음
	- ‘푸크시아'가 당초 임베디드 시스템 위한 OS로 개발 되다가, 
	태블릿PC와 스마트폰은 물론, 사물인터넷 및 ATM기, 
	심지어 신호등을 비롯한 일상의 기기에도 적용이 가능함

2) 스마트카 운영체제
[출처 : https://blogger.pe.kr/422]

- 스마트 자동차 운영체제는 크게 두가지로 나눌수 있다 .
하나는 스마트 차량에 주행에 직접 관여하는 운영체제가 있고 
다른 하나는 주행 기능 외에 운전자의 부가적 편의 기능을 위한 서비스를 주로하는 
'인포테인먼트' 운영체제가 있다.

- 실제 차량의 주행에 사용되는 운영체제는 크게 'OSEK' 와 블랙베리사의 'QNX'가 있다.
인포테인먼트 운영체제는 구글사의 '오토', 애플사의 '카플레이', MS사의 '윈도우 인 더 카'와
리눅스사의 'AGL', 삼성의 '타이젠'등이 있다. 

- 현대차는 커넥티드카에 최적화된 독자적인 차량용 운영체제 
‘ccOS (connec-ted car Operating System)’를 개발하고 있음

# 운영체제의 기능
- 운영체제 기능은 크게 [자원 관리 기능], [시스템 보호], [네트워킹], [명령 해석기]와 [시스템 관리]으로 구분된다.
- 자원 관리 기능에는 [메모리 관리, 프로세스 관리, 주변장치 관리, 파일 관리] 4가지가 있다.
- 운영체제는 파일 사용 권한 부여하고 데이터 암호화 등 서비스를 제공하며 데이터와 시스템을 보안한다.
- 프로세서는 다양한 방법으로 구성된 네트워크 이용, 완전 접속과 부분 접속 방법으로 연결한다.


# 운영체제 서비스
- 부트스트래핑 서비스는 운영체제를 메인 메모리에 적재하는 과정이다.
- [사용자 서비스]에는 사용자 인터페이스(명령어), 메뉴 인터페이스, GUI(Graphical User Interface), 
프로그램 실행, 입출력 동작 수행, 파일 시스템 조작, 통신(네트워크), 오류 탐지가 있다.
- [시스템 서비스]에는 자원 할당, 계정, 보호와 보안이 있다.

# 운영체제의 미래
- 사람 같은 인공지능 OS 즉, OS가 기존의 단순 정보전달 차원에서 문제 해결능력을 갖추는 단계까지 이르렀다.
- 새로운 운영체제는 태블릿PC와 스마트폰은 물론, 사물인터넷 및 ATM기, 심지어 신호등을 비롯한 일상의 기기에도 적용이 가능하다.
- 애플과 구글, 마이크로소프트(MS) 등 주요 기업이 스마트카에 탑재하는 운영체제(OS)를 선보이고 있다.
